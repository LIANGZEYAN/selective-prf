==============================================================================
  EXPERIMENT DETAILS: Pairwise LLM Ranking for PRF Preference Prediction
==============================================================================

------------------------------------------------------------------------------
1. RESEARCH QUESTION
------------------------------------------------------------------------------
Can a large language model (LLM), used as a pairwise document ranker, predict
whether Pseudo-Relevance Feedback (PRF) is beneficial or harmful for a given
query — without access to user interaction data?

------------------------------------------------------------------------------
2. DATASET
------------------------------------------------------------------------------
Source:
  A user study with 43 queries. For each query, participants were shown a
  search result page constructed by Team-Draft Interleaving (TDI) of documents
  from two systems: a Baseline retriever (A) and a PRF retriever (B).

Files:
  result_interleaved_with_text.csv  — 387 rows (43 queries × 9 documents)
  preference.csv                    — 43 rows of human preference labels

Document origin labels (origin_label column):
  A-Only        — retrieved only by Baseline (discriminative)
  B-Only        — retrieved only by PRF       (discriminative)
  Both          — retrieved by both systems   (non-discriminative)
  Easy-Negative — retrieved by neither        (non-discriminative)

Human preference label derivation (preference column):
  Each query has a b_preference_ratio = B-Only clicks / (A-Only + B-Only clicks),
  i.e. the fraction of discriminative clicks that went to PRF documents.
  Original labels (from user study):
    PRF-Benefit       : b_preference_ratio > 0.60
    PRF-Hurt          : b_preference_ratio < 0.40
    Neutral           : 0.40 <= ratio <= 0.60
    Insufficient-Data : no discriminative clicks (2 queries, excluded)

Label distribution (original):
  Neutral           : 21 queries
  PRF-Hurt          : 11 queries
  PRF-Benefit       :  9 queries
  Insufficient-Data :  2 queries  (excluded from all evaluation)

------------------------------------------------------------------------------
3. METHOD
------------------------------------------------------------------------------
Framework : ielab/llm-rankers  (https://github.com/ielab/llm-rankers)
            PairwiseLlmRanker with heapsort, k=9

Models tested:
  (A) Qwen/Qwen2.5-7B-Instruct   — run locally on GPU (CUDA)
  (B) Qwen/Qwen2.5-72B-Instruct  — run via OpenAI-compatible API
                                    (http://api.llm.apps.os.dcs.gla.ac.uk/v1)

Pairwise prompt:
  Given a query "{query}", which of the following two passages is more
  relevant to the query?

  Passage A: "{doc1}"
  Passage B: "{doc2}"

  Output Passage A or Passage B:

Sorting algorithm : heapsort  (O(n + k*log n) comparisons)
  Each comparison fires both the forward and reverse prompt (position-bias
  correction): if forward says A and reverse says B, document 1 wins;
  otherwise document 2 wins or a tie is declared.
  Average comparisons per query: ~21 (7B) and ~31 (72B).

Truncation:
  Query    : 32 tokens  (~128 characters)
  Passage  : 128 tokens (~512 characters)

DCG-weighted preference ratio:
  After reranking, a PRF-preference ratio r_q is computed as:

      G_PRF  = sum of gain(rank_i) for all B-Only documents
      G_Base = sum of gain(rank_i) for all A-Only documents
      gain(rank) = 1 / log2(rank + 1)          [standard DCG gain]

      r_q = G_PRF / (G_PRF + G_Base)

  Documents labelled Both or Easy-Negative are ignored (non-discriminative).
  r_q = NaN if no discriminative documents exist (Insufficient-Data).

  Intuition: if the LLM ranks PRF-only documents higher than Baseline-only
  documents, r_q > 0.5, indicating predicted PRF benefit.

------------------------------------------------------------------------------
4. THRESHOLD DERIVATION  (h = 0.061)
------------------------------------------------------------------------------
The LLM produces a continuous ratio r_q in [0, 1]. To convert this to a
three-way label (PRF-Benefit / Neutral / PRF-Hurt) we define a neutral band:

    PRF-Benefit       : r_q > 0.5 + h
    PRF-Hurt          : r_q < 0.5 - h
    Neutral           : 0.5 - h <= r_q <= 0.5 + h

The threshold h = 0.061 is derived directly from the model's own prediction
distribution, using the Median Absolute Deviation (MAD):

    MAD = median( |r_q - median(r_q)| )  over all 41 queries with valid ratios

For Qwen-2.5-72B-Instruct:
    median(r_q) = 0.5396
    MAD         = 0.0615  ≈ 0.061

This is a robust, data-driven criterion: the neutral band [0.439, 0.561]
captures predictions that are within one MAD of the median — i.e., where the
model expresses no strong directional preference. Queries outside this band
are those where the model's ranking signal is meaningfully above or below
the typical uncertainty level.

The MAD is preferred over the standard deviation because it is resistant to
outliers in the ratio distribution and does not require distributional
assumptions.

Note: the same h=0.061 is applied when re-labelling the human ground truth
(b_preference_ratio), so that the evaluation focuses on queries where the
human preference signal is also unambiguous.

For Qwen-2.5-7B-Instruct, the MAD = 0.021 — reflecting that the 7B model
produces a tightly clustered distribution near 0.5, indicating it lacks
sufficient discriminative power for this task.

------------------------------------------------------------------------------
5. EVALUATION PROTOCOL
------------------------------------------------------------------------------
Neutral queries are EXCLUDED from evaluation.
Rationale: for queries where PRF has negligible impact (Neutral), whether the
system correctly identifies them is not operationally meaningful. The research
question concerns detection of clear benefit or harm.

At h=0.061, the evaluation set comprises:
  #PRF-Hurt    : 14 queries
  #PRF-Benefit : 12 queries
  #Neutral (excluded) : 15 queries

Metrics:
  Hurt Recall    = correctly predicted PRF-Hurt / total PRF-Hurt
  Benefit Recall = correctly predicted PRF-Benefit / total PRF-Benefit
  Accuracy (B+H) = (Hurt correct + Benefit correct) / (Hurt + Benefit)

  All three metrics are reported with 95% Wilson score confidence intervals,
  which are appropriate for proportions and well-behaved with small samples.

  Pearson and Kendall-tau correlation are NOT used as primary metrics because
  this is a classification task (benefit vs. hurt), not a ranking task.

Key findings:
  1. Qwen-2.5-72B significantly outperforms Qwen-2.5-7B on Hurt Recall
     (0.643 vs 0.143), confirming that model scale is critical for this task.

  2. The 7B model produces ratios tightly clustered near 0.5 (MAD=0.021),
     meaning it cannot reliably distinguish PRF-Hurt from PRF-Benefit queries.
     The 72B model has a much wider distribution (MAD=0.061), demonstrating
     genuine discriminative signal.

  3. Benefit Recall is identical (0.833) for both models — reflecting that
     the slight PRF-positive bias of both models happens to align with the
     benefit cases regardless of scale.

  4. Results are stable across a range of h values (Table 2), supporting the
     robustness of the approach. The improvement from h=0.00 to h=0.06 is
     monotonic, and using h=0.061 (the MAD) provides the principled stopping
     criterion.

  5. Confidence intervals are wide due to small sample size (n=26 at h=0.061),
     which is a direct consequence of the scale of the user study. This is a
     known limitation and is reported transparently.

------------------------------------------------------------------------------
7. AGREEMENT METRICS (LLM vs Human — "can LLM replace users?")
------------------------------------------------------------------------------
These metrics assess whether the LLM's continuous ratio r_q moves in the same
direction as human preference, independent of any classification threshold.
They address the question: does the LLM exhibit the same behaviour pattern as
real users, so that it could substitute for costly user studies?

Three complementary metrics are reported:

  AUC (c-statistic)
    Binary: human PRF-Benefit vs PRF-Hurt queries only (n=20; nB=9, nH=11).
    Interpretation: probability that the LLM assigns a higher ratio to a
    randomly chosen Benefit query than to a randomly chosen Hurt query.
    0.5 = random, 1.0 = perfect discrimination. Threshold-free.
    95% CI via Hanley-McNeil standard error.

  Goodman-Kruskal Gamma (γ)
    Ordinal: human 3-way label (Hurt < Neutral < Benefit) vs LLM ratio.
    All 41 valid queries included. γ = (C − D) / (C + D), where tied pairs
    are EXCLUDED from the denominator. Less strict than Kendall-τ (which
    dilutes the score by counting ties as non-concordant).

  Spearman ρ
    Continuous: LLM ratio vs human b_preference_ratio (all 41 queries).
    Measures monotone rank agreement on the raw continuous signals.
    Less strict than Pearson (no linearity assumption).

Table 3: Agreement metrics — LLM ratio vs human preference signal

Model                    AUC      [95% CI]          Gamma    Spearman ρ   p
-----------------------------------------------------------------------------
Qwen-2.5-7B-Instruct     0.505   [0.244, 0.766]    0.020    0.024        0.88
  (local, GPU)

Qwen-2.5-72B-Instruct    0.788   [0.578, 0.998]    0.329    0.400        0.010
  (API)

Key findings:
  1. Qwen-2.5-7B shows no agreement with human preferences across all three
     metrics: AUC ≈ 0.5 (random), Gamma ≈ 0, Spearman ρ ≈ 0 (p=0.88).
     The 7B model cannot replicate human pairwise preference judgements.

  2. Qwen-2.5-72B shows meaningful agreement: AUC=0.788 (the LLM picks the
     correct query as Benefit over Hurt nearly 79% of the time),
     Gamma=0.329 (moderate ordinal concordance across all three classes),
     Spearman ρ=0.400 (p=0.010, significant at 1%).

  3. The lower bound of the 72B AUC CI (0.578) comfortably exceeds chance
     (0.5), supporting the claim that the LLM provides genuine signal about
     user preference even at this sample size.

  4. All three metrics converge on the same conclusion: scale is decisive.
     A 72B model is a plausible (though imperfect) proxy for user judgement;
     a 7B model is not.
